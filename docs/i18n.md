# i18n — Đa ngôn ngữ

## 1. Phạm vi & ngôn ngữ

5 ngôn ngữ hỗ trợ chính thức:

| Code | Ngôn ngữ | Mặc định khu vực |
|---|---|---|
| `en` | English | US |
| `vi` | Tiếng Việt | VN — **default** |
| `ja` | 日本語 | JP |
| `zh` | 中文 (giản thể) | CN |
| `ko` | 한국어 | KR |

Không hỗ trợ chính thức tiếng khác — chỉ thêm khi tổ chức nâng gói "Multi-locale Plus".

## 2. Tầng frontend

- Thư viện: `next-intl` (App Router).
- Cấu trúc:
```
packages/i18n/
└── messages/
    ├── en.json
    ├── vi.json
    ├── ja.json
    ├── zh.json
    └── ko.json
```
- Mỗi app Next.js:
```
apps/<portal>/src/i18n/
├── routing.ts          # locale prefix
├── request.ts          # next-intl config
└── middleware.ts       # detect locale
```
- Format: ICU MessageFormat (cho plural / select).
- Quy tắc đặt key: `<feature>.<screen>.<element>.<role?>` ví dụ `booking.create.title`, `customer.list.column.name`.
- Locale mặc định: `vi`. Fallback chain: `vi → en`.
- URL có prefix: `/vi/booking`, `/en/booking`, … (trừ `client-web` có thể auto detect theo Accept-Language).

## 3. Tầng backend

- Mỗi user có `locale` ở bảng `users`. Token JWT carry `loc` claim.
- Các trường `name`, `description` ở dữ liệu tenant lưu dạng `JSONB`:
  ```json
  {"vi": "Triệt nách", "en": "Underarm hair removal", "ja": "脇脱毛"}
  ```
- API trả về theo locale request (header `Accept-Language` hoặc `?locale=`); fallback `vi → en` nếu thiếu.
- Notification template lưu theo `(template_code, locale)`; thiếu locale fallback `en`.
- Lỗi từ backend dùng `error_code` (vd `BOOKING_CONFLICT_SLOT`); FE map sang i18n. Message tiếng Việt mặc định kèm theo cho debug.

## 4. Định dạng

- Số: `Intl.NumberFormat(locale, { style: 'currency', currency: org.currency || 'VND' })`.
- Ngày: `dayjs.locale(locale)` + format theo region (vd `vi`: `DD/MM/YYYY`, `en`: `MM/DD/YYYY`, `ja`: `YYYY/MM/DD`).
- Múi giờ: theo `branch.timezone` ở backend, không theo locale.

## 5. Quy trình thêm nội dung

1. Khi thêm chuỗi UI mới: sửa cả 5 file `messages/*.json`. Nếu chưa có dịch chuẩn, dùng key tiếng Anh và đánh dấu `// TODO i18n`. Không hardcode.
2. CI lint: script `tools/i18n-check.ts` quét key trong code vs file json, fail nếu thiếu.
3. Khi thêm dữ liệu seed (vd dịch vụ) → cung cấp ít nhất `vi` + `en`. Các locale khác có thể auto-translate qua AI service nếu org bật cờ `auto_translate`.

## 6. Auto-translate (M4+)

- `ai-service` có endpoint `POST /i18n/translate` nhận `{key, source_locale, source_text, target_locales[]}` → trả về bản dịch + lưu cache.
- Reviewer của tổ chức duyệt trước khi publish.

## 7. Test

- Snapshot test cho mỗi locale của các trang chính.
- Test phụ trợ: ép `process.env.NEXT_PUBLIC_LOCALE=ja` chạy E2E smoke.
