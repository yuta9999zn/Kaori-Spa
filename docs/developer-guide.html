4. AuraDB — Data Layer

Module shared/data-store.js là lớp dữ liệu dùng localStorage để giữ dữ liệu giữa các màn hình trong cùng một phiên làm việc.

localStorage Keys

KeyKiểu dữ liệuMô tả
aura_transactionsTransaction[]Mỗi thanh toán hoàn tất
aura_inventory_movesInventoryMove[]Nhập/xuất kho
aura_daily_entriesDailyEntry[]Ghi thủ công (tồn quỹ, chi phí)
aura_combosCombo[]Combo buổi + gói dịch vụ
aura_combo_sessionsComboSession[]Từng buổi được dùng trong combo

Transaction Schema

// Ghi khi thanh-toan.html hoàn tất { id: "tx_1713097200000", date: "2026-04-14", // YYYY-MM-DD type: "dv", // "dv" | "mp" method: "tm", // "tm" | "the" | "ck-loc" | "ck-cty" amount: 680000, // VND bookingId: "BK-10425", customer: "Nguyễn Thị Hoa", service: "Swedish Massage", createdAt: 1713097200000 // timestamp }

InventoryMove Schema

// Ghi khi nhap-kho.html hoặc xuat-kho.html submit { id: "mv_1713097200001", productId: "NB-001", // mã NB từ báo cáo Excel delta: +10, // dương = nhập, âm = xuất type: "nhap", // "nhap" | "xuat" note: "Spa Supply / INV-001", receipt: "INV-001", createdAt: 1713097200001 }

Combo Schema (xem Section 5)

// Combo buổi (1 dịch vụ × N lần) { id: "cb_001", kind: "session", // "session" | "package" customerId: "24D11001", serviceId: "nach", // dịch vụ duy nhất totalSessions: 10, unitPrice: 200000, purchaseDate: "2026-01-15", expiryDate: "2027-01-15" // tuỳ chọn } // Gói combo (nhiều dịch vụ khác nhau) { id: "cb_002", kind: "package", customerId: "24D11001", packageId: "GOI-BRIDAL", // từ goi-dich-vu.html items: [ // các dịch vụ trong gói { serviceId:"nach", sessions:10, unitPrice:200000 }, { serviceId:"chan", sessions:5, unitPrice:300000 }, { serviceId:"facial",sessions:3, unitPrice:350000 }, ], purchaseDate: "2026-01-15" }

AuraDB API

SET addTransaction(tx)
Gọi từ thanh-toan.html khi thanh toán xong
Tự tạo id + createdAt
GET getTransactions(date?)
Lấy danh sách giao dịch, lọc theo ngày
date: "YYYY-MM-DD" hoặc omit cho tất cả
GET getMonthlyReport(y, m)
Tổng hợp DT, phân loại, theo ngày
→ { dv, mp, total, byDay, byMethod }
SET addInventoryMove(mv)
Gọi từ nhap/xuat-kho khi submit
delta dương/âm
GET getStock(productId, base)
Tồn kho = base + tổng delta moves
base: qty mặc định trong INVENTORY[]
SET addCombo(combo)
Tạo combo mới (buổi hoặc gói)
kind: "session" | "package"
SET recordComboSession(comboId, svcId?)
Ghi 1 buổi đã dùng trong combo
svcId: bắt buộc với kind=package
DEL clearAll()
Xoá hết localStorage (dev only)
⚠️ Không dùng trên production

5. Combo & Liệu Trình — 2 Loại

A Combo Buổi (Liệu Trình)

Khách mua 1 dịch vụ cụ thể theo số buổi. Mỗi lần đến sử dụng dịch vụ đó, ghi nhận 1 buổi.

Ví dụ:

Khách A mua 10 buổi triệt nách × 200k/buổi

→ Đã dùng 8 buổi, còn 2 buổi (400k)

→ Mỗi lần đến, tạo booking chọn service "Nách" và gán combo này

Schema key:

kind: "session", serviceId: "nach", totalSessions: 10
→ Màn hình: bao-cao/lieu-trinh.html

B Gói Combo (Nhiều Dịch Vụ)

Khách mua 1 gói bao gồm nhiều dịch vụ khác nhau. Mỗi dịch vụ trong gói có số buổi và giá riêng.

Ví dụ:

Gói Bridal = 10 nách + 5 chân + 3 facial

→ Khách đến làm 1 dịch vụ → chọn gói → chọn dịch vụ → ghi buổi

→ Hệ thống trừ buổi của đúng dịch vụ đó

Schema key:

kind: "package", items: [{serviceId, sessions, unitPrice}]
→ Màn hình: dich-vu/goi-dich-vu.html
